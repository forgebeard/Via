# Журнальный движок и путь `send_safe`

## Планировщик

При `JOURNAL_ENGINE_ENABLED=1` в `cycle_settings` основной цикл `check_all_users` вызывает только **журнальный тик** (`run_journal_tick`): обработка задач через `journal_handlers` и именованные шаблоны `tpl_*`.

Цикл **`check_user_issues`** (поле `processor` / уведомления по статусу, просрочке, журналу без v2) в этом режиме **не** выполняется.

## Исключения

Отдельная задача **`check_unassigned_new_issues`** продолжает вызывать **`send_safe`** для новых неназначенных задач. Тело сообщения собирается тем же **`build_matrix_message_content`**, что и для назначенных пользователей: через `tpl_new_issue` и сессию БД.

Итог: при включённом журнале визуальный стиль **unassigned new** должен совпадать с настройками тех же `tpl_*`, что и в остальной системе v2.
