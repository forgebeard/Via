# Аудит и логирование

Описание каналов логирования Via и настроек аудита действий администратора.

## Каналы логирования

| Канал | Источник | Куда пишет | Назначение |
|-------|----------|------------|------------|
| Журнал бота | `bot` | stdout контейнера + таблица `events` | Циклы опроса Redmine, отправка уведомлений, ошибки |
| Журнал админки | `admin` | stdout контейнера | HTTP-запросы, ошибки приложения |
| Аудит действий | `admin` | таблица `events` (метка `[ADMIN]`) | Вход/выход, управление ботом, CRUD-операции |

Все контейнерные логи доступны через `docker compose logs`.

## Просмотр в панели

**События** (`/events`) — единый журнал записей из таблицы `events`:

- **Фильтрация** по диапазону дат.
- **CSV-экспорт** — кнопка на странице.
- Записи админки помечены `[ADMIN]`, записи бота — без метки.

## Настройки (`.env`)

| Переменная | Умолчание | Описание |
|------------|-----------|----------|
| `ADMIN_EVENTS_LOG_CRUD` | `0` | `1` — писать в `events` каждую CRUD-операцию (создание / изменение / удаление пользователей, групп, маршрутов) |
| `BOT_LOG_LEVEL` | `INFO` | Уровень логирования бота: `DEBUG`, `INFO`, `WARNING`, `ERROR` |

### Когда включать `ADMIN_EVENTS_LOG_CRUD`

- **`0` (по умолчанию)** — достаточно для большинства установок. В журнал попадают входы, выходы и управление ботом.
- **`1`** — полный аудит. Полезно при нескольких администраторах или требованиях комплаенса. Увеличивает объём записей в `events`.

## Формат записи в `events`

```
2025-01-15 10:32:07 | [ADMIN] Пользователь admin: создан пользователь "ivanov" (id=42)
2025-01-15 10:33:12 | Бот: отправлено уведомление в !room:matrix.local (issue #1024)
```

Поля таблицы `events`:

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | `serial` | PK |
| `timestamp` | `timestamptz` | Время события |
| `source` | `text` | `bot` или `admin` |
| `message` | `text` | Текст записи |

## Ретенция

Автоматической очистки нет. При необходимости:

```bash
# Удалить записи старше 90 дней
docker compose exec postgres psql -U bot -d via \
  -c "DELETE FROM events WHERE timestamp < now() - interval '90 days';"
```

Для автоматизации — добавьте команду в cron хоста или используйте pg_cron.

## Внешний сбор логов

Stdout контейнеров совместим с любым log-драйвером Docker:

```yaml
# docker-compose.override.yml
services:
  bot:
    logging:
      driver: syslog
      options:
        syslog-address: "udp://logserver:514"
        tag: "via-bot"
  admin:
    logging:
      driver: syslog
      options:
        syslog-address: "udp://logserver:514"
        tag: "via-admin"
```

Альтернативы: `journald`, `fluentd`, `gelf`.
